# Licensing policy

A bundle is a mirror of a website's content. Publishing it here redistributes
that content, so the registry accepts a bundle **only** when redistribution is
permitted:

- **You own the source** (your own site / docs), or
- **The content is under a license that permits redistribution** — e.g.
  CC-BY, CC-BY-SA, CC0 / public domain, or the documentation of an
  open-source project under a permissive/copyleft license (MIT, Apache-2.0,
  GPL family, etc.).

Requirements for every entry:

- `license` — the SPDX identifier (or clear name) of the **content's** license,
  not okf-kit's.
- `source_url` — where the content came from, so licensing can be verified.
- Attribution and share-alike obligations (e.g. CC-BY-SA) are satisfied by the
  bundle: each concept file cites its `source_url`, and the entry records the
  license.

Bundles whose licensing can't be verified are declined. When in doubt, publish
your own content, or link the source instead of mirroring it.
